Container and Gridlines

It is important to understand the relationship between the parent container and it's child elements. If you have worked with Flexbox, you should already be familiar with this.

Container

The parent element. You will be stting grid dimensions here. This of the container as a blueprint, where you will assign lengths and dimensions.

Child Elements

Here you will assign to each element how much of the grid the respective element will take up.

Setting the Dimensions of Our Grid

First things first, lets create our parent container using a div tag and give it display: grid; CSS. By assigning the div to have a display property of grid, you are preparing the browser to receive your grid commands.

Looking at our example on the first page, we can see that there are three rows. We can assign the rows a length using the CSS property grid-template-rows. For reference, our heights for the header and footer are 75px and the height of our body is 300px. Using the CSS command, you declare the length of the rows individually so the end result looks like this:

grid-template-rows: 75px 300px 75px;

The same syntax works with columns, utilizing the grid-template-columns. You can see from the picture that there's only one instance where we split into columns, so we should only need two rows. Remember that with width (say that 10 times fast), we try to use percentages where applicable. In our case, the percentages are 1/5th and 4/5th of the page. The CSS will look like this.

grid-template-columns: 20% 80%;

And with that, we've made our grid! If you were to render this in the broswer and inspect it using DevTools, you will notice that a grid appear on the page when you select that container. Awesome.
